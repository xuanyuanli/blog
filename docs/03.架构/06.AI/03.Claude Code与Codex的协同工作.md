当然，日常面临的环境是复杂的，我推荐的用法是：
- 对于新项目或新功能开发，我推荐的用法是：让Codex写产品需求、架构方案，Claude Code写具体的代码实现。编码完成之后，再交给Codex做代码Review，把Review方案交回给Claude Code，让他评价并给出具体修复方案，审阅之后由Cluade Code执行。
- 对于Bug修复，我推荐的用法是：让Codex深度分析，给出修复方案。把同样的提示词和Codex给出的修复方案交给Claude Code来评审，给出具体修复方案，审阅之后由Cluade Code执行。
